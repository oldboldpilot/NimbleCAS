# `nimblecas.quantum` — Reference

**Author:** Olumuyiwa Oluwasanmi

Source: `src/quantum/quantum.cppm`

An **exact, purely symbolic non-commutative operator algebra** over the
Gaussian-rational scalars [`Complex`](complex.md) (`Q + Qi`, ROADMAP §7.15). It
provides the algebraic substrate physicists reach for first: named operator
symbols that do **not** commute, Lie brackets `[A,B] = AB − BA`, anticommutators
`{A,B} = AB + BA`, Dirac bra/ket objects, and the dagger (adjoint) involution.
Operators are trees of `std::variant` nodes shared through `CowPtr` (the same
copy-on-write model as [`Expr`](symbolic.md)), so copies are O(1) and safe to
read across threads.

This module is **self-contained**: it defines its own small COW type `Op` over
its own operator variant `OpNode`, and deliberately does **not** touch the core
[`symbolic`](symbolic.md) `ExprNode` variant (that would ripple through the ~40
modules that depend on it). It mirrors symbolic's COW / `std::variant` style
rather than extending it.

```cpp
import nimblecas.quantum;
```

Depends on [`core`](core.md) (`CowPtr`, `Result`/`MathError`),
[`ratpoly`](ratpoly.md) (`Rational`, exposed via `Complex::real()`/`imag()`),
and [`complex`](complex.md) (the exact scalar coefficients).

## Honesty boundary — what this is, and what it is not

**It IS** exact symbolic manipulation of an associative `*`-algebra generated by
non-commuting symbols with exact `Complex` (Gaussian-rational) coefficients.
Operator order is preserved; scalars are exact; the imaginary unit `i` in a
relation such as `[x,p] = i·ℏ` is represented **exactly** (`Complex::i()`), never
as a float.

**It is NOT a numerical quantum-mechanics solver.** There is deliberately:

- **no Hilbert-space representation** — no wavefunctions, no `L²` inner-product
  integral;
- **no matrix representation** of operators, no eigenvalue / spectral solving;
- **no time evolution, no measurement, no tensor products** of state spaces;
- **no physical constants baked in** — `ℏ`, masses, frequencies, etc. are just
  symbols the caller supplies.

**Canonical commutation relations are rewrite inputs the caller supplies, not
physics baked into the engine.** To impose `[x,p] = i·ℏ`, the caller normal-forms
an expression and then substitutes/interprets the residual `[x,p]` monomial. What
the engine guarantees on its own is the purely algebraic facts — e.g.
`[x,p] = −[p,x]` and `[A,A] = 0` — plus faithful bookkeeping of whatever relation
the caller layers on top.

**Overflow precondition on `dagger`.** Scalar coefficients are the
overflow-checked Gaussian rationals of [`complex`](complex.md), so `normal_form`,
`add`, `multiply`, and `scale` surface `MathError::overflow` through the `Result`
railway rather than wrapping. `dagger`, however, is **total** (it returns a plain
`Op`, not a `Result`): its only theoretically-fallible step is conjugating a
scalar's imaginary part, which is unrepresentable **only** when that part is
exactly `INT64_MIN` (whose negation overflows). That is a documented
**precondition** — callers must not build scalar coefficients with an `INT64_MIN`
numerator or denominator part (the same `int64` boundary the whole `Rational`
layer carries). On such a degenerate input `dagger` leaves the part unchanged
(a saturating no-op) rather than fabricating an unrepresentable value; every
non-degenerate scalar conjugates exactly.

## `class Op` — a copy-on-write operator-algebra expression

A copy-on-write handle to an immutable operator-algebra expression. Value
semantics with O(1) copies: the payload is logically immutable and shared through
`CowPtr`, so an `Op` is cheap to pass around and safe to read across threads.
Every `Op` memoizes, at construction, its subtree node count (`size_`) and a
structural hash (`hash_`), so `size()` and `structural_hash()` are O(1).

### Leaf factories

| Factory | Signature | Notes |
| :--- | :--- | :--- |
| `symbol` | `static auto symbol(std::string name) -> Op` | A named non-commutative operator symbol `A` (not self-adjoint, not the dagger form). |
| `self_adjoint_symbol` | `static auto self_adjoint_symbol(std::string name) -> Op` | A symbol declared self-adjoint (Hermitian): `dagger(A) == A`. |
| `make_symbol` | `static auto make_symbol(std::string name, bool adjoint, bool self_adj) -> Op` | General symbol factory used by `dagger`: `adjoint` selects the `A†` form, `self_adj` marks it Hermitian (in which case `adjoint` is always forced false). |
| `scalar` | `static auto scalar(Complex value) -> Op` | The scalar `c` times the identity — a `c·I` element of the algebra. |
| `identity` | `static auto identity() -> Op` | The identity operator `I`. |
| `ket` | `static auto ket(std::string name) -> Op` | The Dirac ket `\|name>`. |
| `bra` | `static auto bra(std::string name) -> Op` | The Dirac bra `<name\|`. |

### Compound factories (order-preserving)

| Factory | Signature | Notes |
| :--- | :--- | :--- |
| `sum` | `static auto sum(std::vector<Op> terms) -> Op` | `A + B + …` (addition commutes). |
| `product` | `static auto product(std::vector<Op> factors) -> Op` | `A·B·…` — **non-commutative**, factor order preserved. |
| `scaled` | `static auto scaled(Complex coeff, Op operand) -> Op` | `coeff · operand`. |

These build raw (unsimplified) tree nodes. Run [`normal_form`](#canonicalisation-and-equality)
to canonicalise.

### Queries

| Method | Signature | Behavior |
| :--- | :--- | :--- |
| `node` | `auto node() const -> const OpNode&` | The underlying variant node (for visiting). |
| `is_equivalent_to` | `auto is_equivalent_to(const Op& other) const -> bool` | Structural (syntactic) equality — the trees are identical. |
| `to_string` | `auto to_string() const -> std::string` | Human-readable rendering, e.g. `A`, `A†`, `I`, `\|psi>`, `<phi\|`, `(A + B)`, `(A B)`, `(3)*A`. |
| `size` | `auto size() const noexcept -> std::size_t` | Memoized subtree node count (O(1)). |
| `structural_hash` | `auto structural_hash() const noexcept -> std::size_t` | Memoized structural hash (O(1)); `a == b` implies equal hashes. |

`operator==(const Op&, const Op&)` is defined as `is_equivalent_to`.

**Structural equality is syntactic:** two `Op`s are equivalent when their trees
are identical. It short-circuits in O(1) when the two handles share the same COW
node, and rejects in O(1) when the memoized hashes differ. This is *not* the
"are these the same operator?" test — for that, canonicalise both sides through
`normal_form` first:

```cpp
normal_form(a).value().is_equivalent_to(normal_form(b).value())
```

`structural_hash()` is consistent with equality: `a.is_equivalent_to(b)` implies
`a.structural_hash() == b.structural_hash()`.

## `OpNode` — the node variant

An `OpNode` wraps a `std::variant` of eight alternatives:

```cpp
struct OpSymbolNode {
    std::string name;
    bool adjoint = false;       // the A† form of the symbol
    bool self_adjoint = false;  // Hermitian: dagger is a no-op (then adjoint is always false)
};
struct OpScalarNode   { Complex value; };            // c, representing c·I
struct OpIdentityNode {};                            // the identity operator I
struct OpKetNode      { std::string name; };         // |name>
struct OpBraNode      { std::string name; };         // <name|
struct OpSumNode      { std::vector<Op> terms; };    // A + B + ...  (addition commutes)
struct OpProductNode  { std::vector<Op> factors; };  // A·B·...  — ORDER PRESERVED
struct OpScaleNode    { Complex coeff; Op operand; }; // coeff · operand

struct OpNode {
    std::variant<OpSymbolNode, OpScalarNode, OpIdentityNode, OpKetNode, OpBraNode,
                 OpSumNode, OpProductNode, OpScaleNode>
        value;
};
```

Visitors over `OpNode` are exhaustive by construction: a `static_assert` in every
`std::visit` chain fails to compile if a new alternative is added without updating
the visitor (mirrors [`symbolic`](symbolic.md)).

## Free-function builders (the public algebra surface)

The ROADMAP-§7.15 named API. `op_symbol`, `op_scalar`, `identity`, `ket`, and
`bra` are thin wrappers over the corresponding `Op` leaf factories.

```cpp
[[nodiscard]] auto op_symbol(std::string name) -> Op;      // A
[[nodiscard]] auto self_adjoint(std::string name) -> Op;   // Hermitian A (dagger fixes it)
[[nodiscard]] auto op_scalar(Complex value) -> Op;         // c·I
[[nodiscard]] auto identity() -> Op;                       // I
[[nodiscard]] auto ket(std::string name) -> Op;            // |name>
[[nodiscard]] auto bra(std::string name) -> Op;            // <name|
```

> **Distinct-generator note.** A symbol's identity **includes its self-adjoint
> flag**, so `op_symbol("H")` and `self_adjoint("H")` are **distinct generators
> that never collapse** — pick one spelling per physical operator and use it
> consistently, or the two will not be treated as equal (even after
> `normal_form`).

### The algebra

```cpp
[[nodiscard]] auto add(const Op& a, const Op& b) -> Op;            // A + B
[[nodiscard]] auto scale(const Complex& c, const Op& a) -> Op;     // c·A
[[nodiscard]] auto multiply(const Op& a, const Op& b) -> Op;       // A·B (order kept)
[[nodiscard]] auto negate(const Op& a) -> Op;                      // -A
[[nodiscard]] auto subtract(const Op& a, const Op& b) -> Op;       // A - B
```

| Function | Behavior |
| :--- | :--- |
| `add` | `A + B`. Flattens one level of an existing sum on either side so trees stay shallow. |
| `scale` | `c·A`, an `OpScaleNode`. |
| `multiply` | `A·B` with **factor order preserved** — multiplication does **not** commute. Flattens one level of an existing product on either side. |
| `negate` | `−A`, built as `scale(−1, A)`. |
| `subtract` | `A − B`, built as `add(A, negate(B))`. |

### Lie brackets

```cpp
[[nodiscard]] auto commutator(const Op& a, const Op& b) -> Op;      // [A,B] = AB - BA
[[nodiscard]] auto anticommutator(const Op& a, const Op& b) -> Op;  // {A,B} = AB + BA
```

`commutator` builds `subtract(multiply(a, b), multiply(b, a))` and
`anticommutator` builds `add(multiply(a, b), multiply(b, a))` — they are plain
compositions of the algebra above, **not** special-cased. The Lie-algebra
identities (antisymmetry, bilinearity, the Leibniz/derivation rule, the Jacobi
identity) are therefore not baked in as bracket-specific rewrite rules; they
**fall out** of `normal_form` applied to these compositions (see the worked
examples below).

### `dagger` — the adjoint involution

```cpp
[[nodiscard]] auto dagger(const Op& a) -> Op;
```

The adjoint/dagger involution, applied recursively. Its rules:

| Input | `dagger` result | |
| :--- | :--- | :--- |
| `A + B` | `A† + B†` | linear over sums |
| `c·A` | `conj(c)·A†` | scalars conjugate |
| `A·B·…·Z` | `Z†·…·B†·A†` | **factor order REVERSED** |
| `A†` (a dagger symbol) | `A` | involution: `(A†)† = A` |
| plain symbol `A` | the distinct `A†` symbol | toggles the dagger flag |
| self-adjoint `H` | `H` | fixed point (`H† = H`) |
| scalar `c` | `conj(c)` | complex conjugate |
| `I` | `I` | |
| `\|k>` | `<k\|` | ket → bra |
| `<b\|` | `\|b>` | bra → ket |

`dagger` is **total** (returns `Op`, not `Result`): see the overflow precondition
in the honesty boundary above. A plain symbol's dagger is a **distinct** `A†`
symbol; only a symbol declared via `self_adjoint`/`self_adjoint_symbol` is a fixed
point.

## Canonicalisation and equality

```cpp
[[nodiscard]] auto normal_form(const Op& a) -> Result<Op>;
```

The canonicalizer. It:

1. **distributes products over sums** on both sides (`A(B+C) → AB + AC`,
   `(A+B)C → AC + BC`), flattening the whole tree into a list of
   coefficient·monomial terms;
2. **folds the exact `Complex` scalars** into each monomial's coefficient;
3. **preserves operator order within each monomial** (the non-commutative core —
   only the intra-monomial factor sequence is meaningful);
4. **collects like ordered monomials**, summing their exact coefficients;
5. **drops zero-coefficient** monomials; and
6. emits a **deterministically-ordered** sum (monomials sorted by a fixed total
   order over their generators).

The result is a canonical tree: **two operators are algebraically equal iff their
normal forms are structurally equal.** So the intended equality/identity test is

```cpp
normal_form(a).value().is_equivalent_to(normal_form(b).value())
```

Because equality reduces to this, every Lie-algebra identity — antisymmetry,
bilinearity, the Leibniz/derivation rule, the Jacobi identity — becomes a
**decidable structural-equality check** on the normalised commutator expressions,
with no bracket-specific logic in the engine. The empty (all-cancelled) normal
form is the zero operator `Op::scalar(Complex{})`.

## Error model

| Function | Fallibility |
| :--- | :--- |
| `normal_form` | Returns `Result<Op>`; propagates `MathError::overflow` from `Complex` coefficient `add`/`multiply` during folding and collection. |
| `add`, `scale`, `multiply`, `negate`, `subtract`, `commutator`, `anticommutator` | Return a plain `Op` — they only **build** trees (no scalar arithmetic yet). Any coefficient overflow surfaces later, when the tree is passed to `normal_form`. |
| `dagger` | **Total** (returns `Op`). Its only fallible step — conjugating a scalar's imaginary part — is overflow-free for every canonical Gaussian rational; the sole unrepresentable case is an `INT64_MIN` part, a documented **precondition** the caller must not violate (the part is then left unchanged). |
| leaf/compound factories, `op_symbol`/`self_adjoint`/`op_scalar`/`identity`/`ket`/`bra` | Total; never error. |

All overflow originates in the [`complex`](complex.md) coefficient layer, which
itself flows through [`ratpoly`](ratpoly.md)'s checked `Rational` arithmetic — an
`int64` numerator or denominator that would wrap surfaces as
`MathError::overflow` rather than silently wrapping.

## Worked examples

```cpp
import nimblecas.quantum;
import nimblecas.complex;
import nimblecas.ratpoly;
using namespace nimblecas;

// c = re + im·i as an exact Gaussian-rational scalar.
auto ci = [](std::int64_t re, std::int64_t im) {
    return Complex::make(Rational::from_int(re), Rational::from_int(im));
};

// "Are these the same operator?" — canonicalise both, then compare structurally.
auto nf_eq = [](const Op& a, const Op& b) {
    auto na = normal_form(a);
    auto nb = normal_form(b);
    return na.has_value() && nb.has_value() && na->is_equivalent_to(*nb);
};
auto is_zero = [&](const Op& a) { return nf_eq(a, op_scalar(Complex{})); };

const Op A = op_symbol("A");
const Op B = op_symbol("B");
const Op C = op_symbol("C");

// Non-commutativity: AB and BA are distinct in general.
nf_eq(multiply(A, B), multiply(B, A));          // false — order matters
// ...but a scalar commutes with everything, and the identity is a unit.
nf_eq(multiply(op_scalar(ci(3, 0)), A),
      multiply(A, op_scalar(ci(3, 0))));          // true  — (cI)A == A(cI)
nf_eq(multiply(identity(), A), A);                // true  — I A == A

// [A,A] = 0, and antisymmetry [A,B] = -[B,A] — both fall out of normal_form.
is_zero(commutator(A, A));                        // true
nf_eq(commutator(A, B), negate(commutator(B, A)));// true

// Jacobi identity: [A,[B,C]] + [B,[C,A]] + [C,[A,B]] = 0.
const Op jac = add(add(commutator(A, commutator(B, C)),
                       commutator(B, commutator(C, A))),
                   commutator(C, commutator(A, B)));
is_zero(jac);                                     // true (not special-cased)

// A canonical pair: [x,p] normal-forms to exactly xp - px, and the imaginary
// unit stays EXACT: i·[x,p] carries coefficients +i and -i (no float ever enters).
const Op x = op_symbol("x");
const Op p = op_symbol("p");
const Op xp_minus_px = add(multiply(x, p), scale(ci(-1, 0), multiply(p, x)));
nf_eq(commutator(x, p), xp_minus_px);             // true — [x,p] = xp - px
const Op scaled = scale(Complex::i(), commutator(x, p));
const Op expect = add(scale(Complex::i(), multiply(x, p)),
                      scale(ci(0, -1), multiply(p, x)));  // +i and -i exactly
nf_eq(scaled, expect);                            // true — i[x,p] = i·xp - i·px

// dagger: order reversal (AB)† = B†A†, involution, and conjugation of i.
nf_eq(dagger(multiply(A, B)), multiply(dagger(B), dagger(A)));  // true
nf_eq(dagger(dagger(A)), A);                                   // true (A†)† = A
nf_eq(dagger(op_scalar(Complex::i())), op_scalar(ci(0, -1)));  // true (iI)† = -iI
nf_eq(dagger(A), A);                                           // false — A† ≠ A

// A declared-Hermitian symbol is a fixed point of dagger; Dirac bra/ket swap.
const Op H = self_adjoint("H");
nf_eq(dagger(H), H);                              // true  — H† = H
nf_eq(dagger(ket("psi")), bra("psi"));            // true  — (|psi>)† = <psi|
nf_eq(dagger(multiply(ket("a"), bra("b"))),
      multiply(ket("b"), bra("a")));              // true  — (|a><b|)† = |b><a|

// Exact rational collection: (1/2)A + (1/2)A = A, with no float rounding.
const Complex half = Complex::from_real(Rational::make(1, 2).value());
nf_eq(add(scale(half, A), scale(half, A)), A);    // true
```
