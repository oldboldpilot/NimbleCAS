# `nimblecas.int128` — Reference

**Author:** Olumuyiwa Oluwasanmi

Source: `src/int128/int128.cppm`

The overflow-checked **middle speed tier**. It exports the compiler-native
signed 128-bit integer `Int128` and its exact fraction `Rational128`, sitting
between the `int64` [`Rational`](ratpoly.md) (magnitude ~`9.2e18`) and the
incoming unbounded arbitrary-precision path (`nimblecas.bigint` /
`BigRational`). `Int128` is `__int128`, which clang-22 lowers to a pair of
64-bit registers on x86-64, so scalar 128-bit add/sub/mul/div stay in registers
and are far cheaper than a heap bignum for values that fit.

**Honesty — this tier is STILL BOUNDED.** Native signed 128-bit spans exactly
`[-2^127, 2^127 - 1]` (roughly `[-1.7e38, +1.7e38]`) — about 18 orders of
magnitude beyond `int64`, but with a hard ceiling. Following the same discipline
as the `int64` `Rational` (Rule 32), an operation whose exact result would leave
the `__int128` range returns `MathError::overflow` rather than silently
wrapping. When a caller sees `overflow` here the answer is **not wrong — it is
out of range**, and the computation should be promoted to the unbounded
`nimblecas.bigint` / `BigRational` tier, which grows without limit at the cost of
heap allocation and slower per-op arithmetic.

**Not vectorised.** AVX/SIMD offers no scalar 128-bit integer multiply or divide
instruction; SIMD only accelerates *batches* of narrower (8/16/32/64-bit) lanes.
This module is deliberately scalar-native — it wins on latency for one-at-a-time
wide values, not on throughput of many narrow ones.

```cpp
import nimblecas.int128;
```

Depends on [`core`](core.md).

## `Int128` free functions

`using Int128 = __int128;` — a first-class arithmetic type on the build target
that simply lacks the standard-library plumbing `int64_t` gets (no
`std::to_string`, no `operator<<`, no portable `std::numeric_limits`). These free
functions supply the pieces the engine needs.

### Conversion and I/O

| Function | Signature | Behavior |
| :--- | :--- | :--- |
| `int128_to_string` | `[[nodiscard]] auto int128_to_string(Int128 v) -> std::string` | Decimal rendering of a signed 128-bit value. Handles the full range including the asymmetric minimum `-2^127` (whose magnitude is unrepresentable as a positive `Int128`) by accumulating through `unsigned __int128`. Total — never fails. |
| `int128_from_string` | `[[nodiscard]] auto int128_from_string(std::string_view s) -> Result<Int128>` | Parse a base-10 signed integer: an optional leading `'+'` or `'-'` followed by one or more decimal digits and nothing else. A non-digit character, a lone sign, or an empty string yields `MathError::syntax_error`; a value outside `[-2^127, 2^127 - 1]` yields `MathError::overflow` (never a wrapped result). Leading zeros are accepted. |
| `int128_from_i64` | `[[nodiscard]] auto int128_from_i64(std::int64_t v) noexcept -> Int128` | Widen a 64-bit integer. Every `std::int64_t` fits, so this cannot fail (`noexcept`). |

### Overflow-checked scalar arithmetic

Each detects a 128-bit overflow via the corresponding `__builtin_*_overflow`
(which accept `__int128` on clang) and returns `MathError::overflow` instead of
wrapping.

```cpp
[[nodiscard]] auto checked_add(Int128 a, Int128 b) -> Result<Int128>;
[[nodiscard]] auto checked_sub(Int128 a, Int128 b) -> Result<Int128>;
[[nodiscard]] auto checked_mul(Int128 a, Int128 b) -> Result<Int128>;
```

| Function | Behavior |
| :--- | :--- |
| `checked_add` | `a + b`, or `MathError::overflow` if the exact sum leaves `[-2^127, 2^127 - 1]`. |
| `checked_sub` | `a - b`, or `MathError::overflow` on wrap. |
| `checked_mul` | `a * b`, or `MathError::overflow` on wrap. |

## `Rational128` — an exact fraction `num/den` in lowest terms over `Int128`

Field-for-field the wider twin of [`nimblecas::Rational`](ratpoly.md). The
canonical form (`den > 0`, `gcd(|num|, den) == 1`, zero represented as `0/1`)
makes equality a member-wise compare and keeps intermediate magnitudes as small
as the 128-bit ceiling allows. Every arithmetic operation is overflow-checked and
returns `Result`; a genuine 128-bit overflow surfaces as `MathError::overflow` so
the caller can promote to `BigRational`.

### Construction

| Constructor / factory | Signature | Behavior |
| :--- | :--- | :--- |
| default | `Rational128()` | The exact zero `0/1`. |
| `make` | `[[nodiscard]] static auto make(Int128 num, Int128 den) -> Result<Rational128>` | Construct `num/den` in canonical form (sign normalised onto the numerator, reduced by gcd). Fails with `division_by_zero` when `den == 0`, or `overflow` when either operand is `int128_min` (`-2^127`), which cannot be sign-normalised without wrapping. |
| `from_int` | `[[nodiscard]] static auto from_int(std::int64_t v) -> Rational128` | The integer `v` as `v/1`. Cannot fail. |

### Accessors

| Method | Signature | Behavior |
| :--- | :--- | :--- |
| `numerator` | `[[nodiscard]] auto numerator() const noexcept -> Int128` | The canonical numerator (carries the sign). |
| `denominator` | `[[nodiscard]] auto denominator() const noexcept -> Int128` | The canonical denominator, always `> 0`. |
| `is_zero` | `[[nodiscard]] auto is_zero() const noexcept -> bool` | `true` when the numerator is `0`. |
| `is_integer` | `[[nodiscard]] auto is_integer() const noexcept -> bool` | `true` when the denominator is `1`. |
| `operator==` | `[[nodiscard]] auto operator==(const Rational128& o) const noexcept -> bool` | Member-wise equality (canonical form makes this exact). |
| `to_string` | `[[nodiscard]] auto to_string() const -> std::string` | Renders `"num"` when integral, else `"num/den"`. |

### Field operations (overflow-checked, exact)

```cpp
[[nodiscard]] auto add(const Rational128& o) const -> Result<Rational128>;
[[nodiscard]] auto subtract(const Rational128& o) const -> Result<Rational128>;
[[nodiscard]] auto multiply(const Rational128& o) const -> Result<Rational128>;
[[nodiscard]] auto divide(const Rational128& o) const -> Result<Rational128>;
[[nodiscard]] auto negate() const -> Result<Rational128>;
[[nodiscard]] auto reciprocal() const -> Result<Rational128>;
```

| Method | Behavior |
| :--- | :--- |
| `add` / `subtract` | `num_/den_ ± o` over the common denominator; each cross product and the combined numerator/denominator flows through `checked_mul` / `checked_add`, so any wrap surfaces as `overflow`. |
| `multiply` | `(num_·o.num_)/(den_·o.den_)`, re-canonicalised. `overflow` if a product wraps. |
| `divide` | `(num_·o.den_)/(den_·o.num_)`, re-canonicalised. Fails `division_by_zero` when `o == 0`; `overflow` if a product wraps. |
| `negate` | `-num_/den_`. Fails `overflow` only when `num_ == int128_min` (`-2^127`), whose magnitude flip is unrepresentable. |
| `reciprocal` | `den_/num_`. Fails `division_by_zero` when `*this == 0`; otherwise cannot overflow (`den_ > 0` and `num_` is never `int128_min`, since `make` rejects that boundary). |

Any operation returns `MathError::overflow` if an intermediate 128-bit product or
sum leaves `[-2^127, 2^127 - 1]`.

## Error model

| Condition | Error |
| :--- | :--- |
| `int128_from_string` on empty input, a lone sign, or a non-digit character | `MathError::syntax_error` |
| `int128_from_string` on a value outside `[-2^127, 2^127 - 1]` | `MathError::overflow` |
| `checked_add` / `checked_sub` / `checked_mul` exact result leaves the 128-bit range | `MathError::overflow` |
| `Rational128::make` with `den == 0` | `MathError::division_by_zero` |
| `Rational128::make` with `num` or `den` equal to `int128_min` (`-2^127`) | `MathError::overflow` |
| `divide` by a zero `Rational128`, or `reciprocal` of zero | `MathError::division_by_zero` |
| Any `Rational128` field op whose intermediate 128-bit arithmetic wraps | `MathError::overflow` |
| `negate` of a `Rational128` whose numerator is `int128_min` | `MathError::overflow` |

`int128_to_string`, `int128_from_i64`, `from_int`, and the accessors
(`numerator` / `denominator` / `is_zero` / `is_integer` / `operator==`) are total
and never error.

## Worked examples

```cpp
import nimblecas.int128;
import nimblecas.core;
using namespace nimblecas;

// Parse a decimal literal beyond the int64 range, then render it back.
auto i128 = [](std::string_view s) { return int128_from_string(s).value(); };
const std::string_view kMax = "170141183460469231731687303715884105727";   //  2^127 - 1
const std::string_view kMin = "-170141183460469231731687303715884105728";  // -2^127

int128_to_string(i128(kMax));             // the INT128_MAX decimal string
int128_to_string(i128("-0"));             // "0"    (negative zero normalises)
int128_to_string(i128("007"));            // "7"    (leading zeros parse)

// Malformed / out-of-range input reports, never wraps.
int128_from_string("12a").error();        // MathError::syntax_error
int128_from_string("").error();           // MathError::syntax_error
int128_from_string("170141183460469231731687303715884105728").error();  // overflow (MAX + 1)

// Overflow-checked scalar arithmetic: 10^18 * 10^18 = 10^36 fits; MAX * 2 does not.
const Int128 e18 = i128("1000000000000000000");
int128_to_string(checked_mul(e18, e18).value());  // "1000000000000000000000000000000000000"
checked_mul(i128(kMax), 2).error();       // MathError::overflow
checked_add(i128(kMax), 1).error();       // MathError::overflow

// Rational128 canonicalisation: reduce and move the sign onto the numerator.
auto r128 = [](Int128 n, Int128 d) { return Rational128::make(n, d).value(); };
r128(2, 4)  == r128(1, 2);                // true  (2/4 reduces to 1/2)
r128(2, -4) == r128(-1, 2);               // true  (sign moves to the numerator)
r128(2, -4).denominator() > 0;            // true  (denominator kept positive)
Rational128::make(1, 0).error();          // MathError::division_by_zero

// Exact beyond int64: (10^18/1) + (1/10^18) = (10^36 + 1)/10^18.
auto sum = r128(e18, 1).add(r128(1, e18)).value();
int128_to_string(sum.numerator());        // "1000000000000000000000000000000000001"
int128_to_string(sum.denominator());      // "1000000000000000000"

// Field laws.
r128(1, 2).add(r128(1, 3)).value()      == r128(5, 6);   // 1/2 + 1/3 = 5/6
r128(2, 3).multiply(r128(3, 4)).value() == r128(1, 2);   // 2/3 * 3/4 = 1/2
r128(2, 3).divide(r128(4, 9)).value()   == r128(3, 2);   // (2/3)/(4/9) = 3/2
r128(3, 7).reciprocal().value()         == r128(7, 3);   // reciprocal(3/7) = 7/3
r128(3, 4).negate().value()             == r128(-3, 4);  // negate(3/4) = -3/4

// Error paths and the int128_min boundary guards (no UB).
r128(1, 2).divide(Rational128{}).error();     // MathError::division_by_zero
Rational128{}.reciprocal().error();           // MathError::division_by_zero
r128(i128(kMax), 1).multiply(Rational128::from_int(2)).error();  // overflow (promote to BigRational)
Rational128::make(i128(kMin), 1).error();     // MathError::overflow (-2^127 rejected as numerator)
Rational128::make(1, i128(kMin)).error();     // MathError::overflow (-2^127 rejected as denominator)
```
