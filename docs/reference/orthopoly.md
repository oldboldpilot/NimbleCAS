# `nimblecas.orthopoly` — Reference

**Author:** Olumuyiwa Oluwasanmi

Source: `src/orthopoly/orthopoly.cppm`

The classical **orthogonal polynomial** families — Chebyshev (first and second
kind), Legendre, Laguerre, and Hermite (physicists' and probabilists') — over
the rationals `Q[x]` (ROADMAP §7.16). Each family is generated by its classical
**three-term recurrence**, evaluated iteratively over
[`RationalPoly`](ratpoly.md) from the `n = 0, 1` base cases. Because the
recurrences involve only exact rational coefficients, the resulting polynomials
are represented **exactly** in `Q[x]` — Legendre and Laguerre genuinely have
fractional coefficients (e.g. `P_2 = (3x^2 - 1)/2`), and this module carries
them without rounding.

```cpp
import nimblecas.orthopoly;
```

Depends on [`core`](core.md) and [`ratpoly`](ratpoly.md).

## The overflow contract

Following the rest of the engine, every step is **exact** and
**overflow-checked** (Rule 32): an `int64` boundary in any `Rational` operation
propagates as `MathError::overflow` rather than wrapping. A negative order is
rejected with `MathError::domain_error`. Internally the variable `x` is the
monomial `RationalPoly::monomial(1, 1)`, so "multiply by `x`" is a
`RationalPoly::multiply` by that monomial.

## API

Each function takes the order `n` and returns `Result<RationalPoly>`; `n < 0`
yields `MathError::domain_error`, and an `int64` coefficient overflow yields
`MathError::overflow`.

| Function | Family | Base cases & recurrence |
| :--- | :--- | :--- |
| `chebyshev_t` | Chebyshev, first kind `T_n` | `T_0 = 1`, `T_1 = x`, `T_{k+1} = 2x·T_k - T_{k-1}`. |
| `chebyshev_u` | Chebyshev, second kind `U_n` | `U_0 = 1`, `U_1 = 2x`, `U_{k+1} = 2x·U_k - U_{k-1}`. |
| `legendre` | Legendre `P_n` | `P_0 = 1`, `P_1 = x`, `(k+1)·P_{k+1} = (2k+1)·x·P_k - k·P_{k-1}`. |
| `laguerre` | Laguerre `L_n` | `L_0 = 1`, `L_1 = 1 - x`, `(k+1)·L_{k+1} = (2k+1 - x)·L_k - k·L_{k-1}`. |
| `hermite_phys` | Hermite, physicists' `H_n` | `H_0 = 1`, `H_1 = 2x`, `H_{k+1} = 2x·H_k - 2k·H_{k-1}`. |
| `hermite_prob` | Hermite, probabilists' `He_n` | `He_0 = 1`, `He_1 = x`, `He_{k+1} = x·He_k - k·He_{k-1}`. |

```cpp
[[nodiscard]] auto chebyshev_t(std::int64_t n)  -> Result<RationalPoly>;
[[nodiscard]] auto chebyshev_u(std::int64_t n)  -> Result<RationalPoly>;
[[nodiscard]] auto legendre(std::int64_t n)     -> Result<RationalPoly>;
[[nodiscard]] auto laguerre(std::int64_t n)     -> Result<RationalPoly>;
[[nodiscard]] auto hermite_phys(std::int64_t n) -> Result<RationalPoly>;
[[nodiscard]] auto hermite_prob(std::int64_t n) -> Result<RationalPoly>;
```

## Error model

| Condition | Error |
| :--- | :--- |
| `n < 0` (every family) | `MathError::domain_error` |
| An `int64` coefficient computation wraps | `MathError::overflow` |

`n = 0` returns the constant `1` for every family; `n = 1` returns the family's
`P_1`/`L_1`/… base case.

## Worked values

Coefficients below are low-degree-first (the `RationalPoly` convention:
`coeffs[i]` is the coefficient of `x^i`).

```cpp
import nimblecas.orthopoly;
import nimblecas.ratpoly;
using namespace nimblecas;

chebyshev_t(2).value();   // 2x^2 - 1        coeffs {-1, 0, 2}
chebyshev_t(3).value();   // 4x^3 - 3x       coeffs {0, -3, 0, 4}
chebyshev_u(2).value();   // 4x^2 - 1        coeffs {-1, 0, 4}

legendre(2).value();      // (3x^2 - 1)/2    coeffs {-1/2, 0, 3/2}
legendre(3).value();      // (5x^3 - 3x)/2   coeffs {0, -3/2, 0, 5/2}

laguerre(2).value();      // (x^2 - 4x + 2)/2 = 1 - 2x + (1/2)x^2   coeffs {1, -2, 1/2}

hermite_phys(2).value();  // 4x^2 - 2        coeffs {-2, 0, 4}
hermite_phys(3).value();  // 8x^3 - 12x      coeffs {0, -12, 0, 8}

hermite_prob(2).value();  // x^2 - 1         coeffs {-1, 0, 1}
hermite_prob(3).value();  // x^3 - 3x        coeffs {0, -3, 0, 1}

chebyshev_t(-1).error();  // MathError::domain_error
```
